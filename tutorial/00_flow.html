Lab 1. UART

In this Lab you will run eshell and get basic knowledge on how to use tool. Follow instruction step-by-step.

You will learn:

  • Start eshell from command-line;
  • Read in (compile) Verilog files;
  • Write simple scripts;
  • Work with databases;
  • Run system commands from tool;
  • Lint modules.

    • Activity:

  • Open console. Move into this tutorial directory (00_flow).
  • Add eshell path into system variable path, ex.:

    a. Windows: set path=%path%;E:\tools\eshell

    b. Linux: Bash export PATH=$PATH:/CAD/eshell

  • Check Lab directory: UART.v + run_01.cmd + run_02.cmd files.
  • Check run_01.cmd content. It just reads UART.v.
  • Launch tool by typing eshell from command prompt.
  • Execute source run_01.cmd
  • Alternatively execute read UART.v. Note that Linux is case sensitive.
  • Try both commands in any sequence. Check disk content into directory where you launched from. There are some additional files eshell*.
  • Check that eshell.edb* files created on launch directory. Eshell.edb keeps information about compiled modules. Eshell.edb00001 keeps compiled module. Eshell.log is log file for current session.
  • Type set_workspace lab1. This will change workspace name from default eshell to lab1. Now log information written to lab1.log and files will be compiled into log.edb*.
  • Read UART.v once again.
  • Clear database lab1 typing clear_db. Return to default database set_workspace eshell. Note that previously created file eshell.log is overwritten when database changed.

  • Remove lab1.log using OS command:

    a. Windows: system del lab1.log

    b. Linux: system rm lab1.log

  • Now execute message NOT. Tool will issue an error. Fix it by typing message NOTE. This command changes level of messages issued by tool. You can run help to get brief information about eshell.
  • Read UART.v file again and check that number of messages increased.
  • To get detailed description about Note/Warning type man ADV-N. Where N â€“ is a number of note.
  • Close tool typing exit
  • Check eshell.log file. It contains information about number of read files and session time.
  • Previous steps were made in interactive mode. To allow batch mode run script can be used. Always on startup eshell executes autosetup.tcl file. Create this one and write following commands:

    message NOTE

    read UART.v

    exit

  • Type eshell from command line and check that tool were startup execute script and exited as specified in autosetup.tcl.
