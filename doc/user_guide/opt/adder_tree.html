Adder Tree Optimization

Сумматоры являются одним из базовых элементов цифровых схем. Они применяются при сложении, вычитании, умножении, сравнении и др. операциях. В отличие от логических операций, сумматоры содержат перемешанные вентили AND, OR, что затрудняет их оптимизацию. Для удаления избыточной логики из групп сумматоров требуется использовать более высокий уровень асбтракции, чем вентильный.

Детектирование логики выполняется на арифметическом уровне по следующему алгоритму.

Этап Пример
Для групп сумматоров для каждого разряда составляется список слагаемых для каждого разряда.

Например, для выражения A + C - B - (D - E) + F таблица для 0-го разряда будет иметь вид:

Для групп сумматоров происходит проверка избыточности для каждого из разрядов, начиная с младшего.

В примере изначально содержатся 4 слагаемых и 2 вычитаемых.

 
 ADD   SUB 
 --------- 
 A0    B0  
 C0    D0  
 E0    -   
 F0    -
Из списка удаляются константные 0.

Например, если D0 = 0, то список преобразуется к виду:

 
 ADD   SUB 
 --------- 
 A0    B0  
 C0    -   
 E0    -   
 F0    -
Проверяется совпадение элементов ADD и SUB.

При совпадении оба элемента удаляются из списка.

Например, если C0 = B0, то список сокращается до 3 записей:

 
 ADD   SUB 
 --------- 
 A0    -   
 C0    -   
 E0    -   
 F0    -
Проводится проверка дублирования слагаемых ADD.

При сложении вида A + A формируется бит переноса, а пара слагаемых удаляется из списка.

Например, если A0 = E0:

 
 ADD   SUB   CARRY 
 ----------------- 
 F0    -     A0
Перед переходом к следующему разряду происходит его дополнение битами CARRY от предыдущего, например для 1-го разряда начальная таблица с учетом бита переноса будет иметь вид:

Операции оптимизации разряда повторяются.

 
 ADD   SUB 
 --------- 
 A1    B1  
 C1    D1  
 E1    -   
 F1    -   
 A0    -

По завершении выполнения алгоритма происходит расчет максимального количества операций сложения/вычитания для каждого разряда. Это число определяет количество сумматоров. Таблицы для каждого разряда дополняются нулями до нужного количества сумматоров. Далее происходит формирование групп сумматоров в виде сбалансированного дерева, т.е. в общем виде сумматор приводится к формуле (A + C) + (F - B) + (E - D).
